# Matsuri Lottery 🎉

ブラウザ上で動作する、シンプルでリッチな抽選アプリケーションです。
ビンゴ大会や各種イベントの抽選会など、様々な場面で活用できます。

🌐 **アプリへのアクセスはこちら**
[https://hyugma.github.io/matsuri-lottery/](https://hyugma.github.io/matsuri-lottery/)

## ✨ 主な特徴

- **派手なアニメーション**: 抽選開始とともに数字が画面内を飛び交い、停止時に当選番号が発表されます。
- **カスタマイズ性**: アプリのタイトル表示やテーマカラーを自由に変更可能です。
- **除外設定**: 途中退出者や既に当選した人など、抽選対象から外したい番号を柔軟に指定できます（例: `1, 2, 5-10`）。
- **履歴管理 & 引換状況のチェック**: 当選履歴が一覧で表示され、番号をクリックすることで「引換済 (🎁)」のマークをつけることができます。番号による検索も可能です。
- **証跡出力**: イベント終了後などに、抽選結果（引換状況を含む）をテキストファイルとして出力（ダウンロード）できます。
- **フルスクリーン対応**: 会場のスクリーンに映すのに最適なフルスクリーン表示が可能です。
- **PWA対応**: スマホやタブレットのホーム画面に追加することで、ネイティブアプリのように全画面で利用可能です。

## 📖 簡単な使い方 (マニュアル)

### 1. 初期設定 (左側の「⚙️ 設定」パネル)
- **タイトル表示**: イベント名など、画面上部に表示するテキストを入力します。
- **テーマカラー**: イベントの雰囲気に合わせてベースカラーを変更します。
- **参加人数**: 抽選対象となる最大人数（番号）を入力し、「更新」を押します。
- **途中退出者 (除外番号)**: 欠席者や抽選から除外したい番号があれば入力し、「追加」を押します。カンマ区切りやハイフンでの範囲指定が可能です（例: `1, 2, 5-10`）。

### 2. 抽選の実行 (中央のメインステージ)
- 準備ができたら **[START]** ボタンを押します。数字が画面を飛び交うアニメーションが始まります。
- タイミングを見計らって **[STOP]** ボタンを押すと、当選番号が発表されます。
- 当選番号は自動的に右側の「当選履歴」に追加され、次回の抽選候補から除外されます。

### 3. 当選履歴の管理 (右側の「🏆 当選履歴」パネル)
- 景品の引き渡しが終わったら、履歴の番号をクリックして **🎁マーク（引換済）** をつけます。
- 大規模な人数の場合は、上の検索ボックスから番号を検索して状態を確認できます。
- 右上の「🔲」ボタンでフルスクリーン表示の切り替えが可能です。

### 4. 終了後の処理 (左側の「⚙️ 設定」パネル)
- **証跡を出力 (txt)**: 抽選の履歴と引換状況をテキストファイルとして保存します。
- **全てリセット**: 全ての履歴と状態を初期化します。

## 🚀 デプロイと設定 (GitHub Pages)

当アプリは GitHub Actions を用いて GitHub Pages に自動デプロイされる構成になっています。
アクセス解析 (Google Analytics) を利用する場合は、以下の手順でトラッキングIDを設定してください。**(※任意です。設定しない場合、解析コードは自動的に除外されます)**

### Google Analytics の設定方法 (オプショナル)
ソースコード内にトラッキングIDを直接記述するのではなく、GitHubのRepository Secretを使用して安全に管理・自動挿入する仕組みになっています。

1. **Secretの追加**:
   - リポジトリの `Settings` > `Secrets and variables` > `Actions` を開きます。
   - `New repository secret` をクリックし、名前に `GA_TRACKING_ID` を指定します。
   - 値にご自身のGoogle AnalyticsのトラッキングID (例: `G-XXXXXXXXXX`) を入力して保存します。

2. **GitHub Pages の設定**:
   - リポジトリの `Settings` > `Pages` を開きます。
   - `Build and deployment` の `Source` を **Deploy from a branch** から **GitHub Actions** に変更します。

これにより、次回 `main` ブランチにPushされた際、自動的にIDが埋め込まれた状態でデプロイされます。
※ `GA_TRACKING_ID` が未設定の場合は、Google AnalyticsのスクリプトはHTMLから完全に削除された状態で公開されます。
